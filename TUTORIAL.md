## 第零课: 全貌 — 数字电源到底在干什么

### 一句话说清

你有一个 24V 输入,想要一个稳定的 12V 输出。不管输入怎么波动、负载怎么变,输出都要死死咬住 12V。

**传统模拟电源**:用电阻电容搭电路来自动调节。
**数字电源**:用单片机来干 — 采样 → 计算 → 调整。

### 像一个人调节水龙头

把电路想象成**水龙头 + 水桶 + 水位计**:

1. 你想让水桶水位保持在 12cm(目标电压)
2. 水位计(ADC)每隔一小会儿就看一眼水位
3. 如果水位低了 → 手(单片机)把水龙头(PWM)拧大一点
4. 如果水位高了 → 手把水龙头拧小一点
5. 不断重复,让水位稳定在 12cm

这就是**闭环控制**的核心:采样 → 计算偏差 → 调整输出。

### 5 个模块各管什么事

| 模块 | 文件 | 类比 |
|------|------|------|
| pwm.c | 水龙头开关 | 快速开合,控制能量多少 |
| adc.c | 水位计 | 把电压/电流读数告诉单片机 |
| pid.c | 大脑 | 算"偏差多少,该拧多大" |
| protection.c | 保险丝 | 出问题立刻切断 |
| main.c | 总调度 | 安排以上所有事按顺序干 |

## 第一课: PWM 模块 — "水龙头开关"

**对应代码文件: Core/Inc/pwm.h + Core/Src/pwm.c**

### 为什么要开关

你把一个 24V 电源直接接到负载上,输出就是 24V。你要 12V 怎么办?

最笨的办法:串一个可变电阻,转一转,调到 12V。问题是电阻会发热,效率极低。

聪明的办法:开关!
- 开关闭合 → 24V 通过
- 开关断开 → 0V 通过
- 快速反复开关 → 平均下来就是 12V

这就叫 **PWM = Pulse Width Modulation(脉冲宽度调制)**。

### 占空比是什么

假设 1 秒开关 100000 次(100kHz):

`
     闭合          断开
   ┌────┐         ┌────┐
   │    │         │    │
───┘    └─────────┘    └────
   ↑    ↑
  0.5us 0.5us
`

**闭合时间 / 总时间 = 占空比。** 50% 占空比 → 24V × 50% ≈ 12V(平均)。

简单记忆:
- 占空比 0.0 = 一直关 = 0V
- 占空比 0.5 = 一半时间开 ≈ 12V
- 占空比 1.0 = 一直开 = 24V(一般不这么做)

### 为什么有两个管子(互补输出)

`
上管开 → 电流从输入流向输出
下管开 → 电流从地流过电感(续流)
`

上管关了下管才开,不能同时开(短路!)。所以一个叫**上管**(接输入),一个叫**下管**(接地)。

### 死区 = 保命时间

`
上管关闭    上管关闭
   ↓          ↓
   ──┐       ┌────
     │       │
─────┘       └──
     └─死区─┘
     上管已关  下管才开
`

这段时间两个管子都关着,叫"死区"。没有死区 → 上下管同时开 → 短路 → 炸管。

### 代码在干什么

`c
// pwm.c 核心函数
void PWM_SetDuty(float duty)
{
    // duty = 0.5 表示 50% 占空比
    // 定时器是个计数器: 0 → 100 → 0 → 100 → ...
    // 计数器值和 CCR 比较,小于就输出高,大于就输出低
    ccr = period * duty;
    __HAL_TIM_SET_COMPARE(&htim_pwm, TIM_CHANNEL_1, ccr);
    // 硬件自动输出 PWM,不需要 CPU 管
}
`

**pwm.h 里的宏定义:**

`c
#define PWM_FREQUENCY    100000UL    // 开关频率 100kHz
#define PWM_PERIOD       720         // 72MHz / 100kHz = 720
#define PWM_DEADTIME_NS  100         // 死区 100ns
#define DUTY_MAX         0.95f       // 最大占空比 95%,留余量防直通
`

### 第一课总结

| 概念 | 人话解释 |
|------|----------|
| PWM | 快速开关来得到想要的电压 |
| 占空比 | 开的时间占多少比例 |
| 互补输出 | 两个管子交替开,效率更高 |
| 死区 | 两个管子都关一小会儿,防止短路 |
| 100kHz | 每秒开关 10 万次 |

## 第二课: ADC 模块 — "眼睛"

**对应代码文件: Core/Inc/adc.h + Core/Src/adc.c**

### 单片机怎么"看到"电压

单片机是数字器件,只认识 0 和 1。但电压是模拟量,可以是 3.14V、2.718V...

**ADC = Analog-to-Digital Converter(模数转换器)**。它的工作:把一个电压值变成单片机认识的数字。

### 打个比方

你用一把尺子量身高。尺子量程 0~100cm,尺子刻度 100 格。身高 175cm,尺子只能量到 100cm → 超量程了。所以用分压:175cm × (10k/100k) = 17.5cm。尺子读到 17.5 → 反推: 17.5 × 10 = 175cm。

### STM32 的 ADC 有多精细

12位 ADC → 把 0~3.3V 分成 **4096** 格:

| 真实电压 | ADC 数字值 |
|----------|-----------|
| 0V | 0 |
| 1.65V | 2048 |
| 3.3V | 4095 |

精度: 3.3V / 4096 ≈ 0.8mV 每格。

**换算公式:**
`
真实电压 = ADC原始值 × (参考电压 / 分辨率)
2048 × (3.3 / 4096) = 2048 × 0.0008059 = 1.65V
`

### 为什么需要分压电阻

你的电源输出 12V,但 ADC 只能测 0~3.3V。直接接 → 烧 ADC。

用两个电阻分压:
`
Vin ---[100k]---+---[10k]---GND
                ↓
              接到ADC

12V × 10k / (100k + 10k) = 12V × 0.0909 = 1.09V (在 ADC 量程内)
`

代码里的 VOUT_DIVIDER_RATIO = 0.1f 就是这个比例。

### 为什么要有均值滤波

开关电源噪声很大:24V 开 → 24V 关 → 24V 开 → 24V 关…每次 ADC 采到的值都不一样,忽高忽低。

解决方案:采 8 次求平均。比如采到:2040, 2050, 2035, 2045, 2038, 2042, 2048, 2042。平均 = 加起来 / 8 = 2042.5。比单次采样稳定多了。

### DMA 是什么?为什么需要它

`
不用 DMA: 每次要采 ADC,CPU 停下手里的事,等 ADC 转换完
          → 像你写作业写到一半,每 5 秒必须去看一眼电饭锅

用 DMA: DMA 是个小助手,它自己看 ADC,看完了放一边
         CPU 忙自己的事,需要时去拿数据就行
         → 像你安心写作业,电饭锅好了助手告诉你
`

### ADC_Process() 在干什么

adc.c 里的 ADC_Process() 做了三件事:

1. **把新采的值存到 FIFO** — FIFO 像一个长度 8 的队列: [旧, 旧, 旧, 新],新数据进来,最早的数据被挤出去
2. **把 8 个值加起来**
3. **除以 8 得到平均值**,然后换算成真实电压

### adc.h 里的关键结构

`c
// 每个通道有原始值、电压值、滤波值
typedef struct {
    uint16_t raw;         // ADC 原始值 0~4095
    float    voltage;     // 换算后的电压(V)
    float    filtered;    // 滤波后的平均值
} ADC_Channel_t;

// 三个通道汇总
typedef struct {
    ADC_Channel_t vout;   // 输出电压
    ADC_Channel_t iout;   // 输出电流
    ADC_Channel_t vin;    // 输入电压
} ADC_Data_t;
`

### 第二课总结

| 概念 | 人话解释 |
|------|----------|
| ADC | 把电压变成数字 |
| 12位分辨率 | 0~3.3V 分成 4096 格 |
| 分压电阻 | 把高电压降到 ADC 能测的范围 |
| 均值滤波 | 采多次求平均,去噪声 |
| DMA | 小助手,自动干苦力活 |

## 第三课: PID 模块 — "大脑"

**对应代码文件: Core/Inc/pid.h + Core/Src/pid.c**

> **这是整个电源控制最核心的部分。如果你只能搞懂一个模块,请搞懂这个。**

### 没 PID 之前是怎样的

开环控制:我给 50% 占空比 → 理论上输出 12V。但实际:负载一变电压就掉,输入波动电压也跟着波动 → 毫无抵抗能力。就像你洗澡调水温:你固定水龙头位置 → 别人开了厨房水 → 你被烫到。

### PID 的思路:看偏差来调整

PID 不做别的事,只问三个问题:

| 项 | 问什么 | 人话 |
|----|--------|------|
| P(比例) | 现在差多少 | 现在是 11.5V,目标 12V,差 0.5V → 拧大一点 |
| I(积分) | 一直差多少 | 一直差 0.5V → 越拧越大,直到差值为零 |
| D(微分) | 变化快不快 | 电压正在快速下降 → 提前多拧一点(电源里通常不用 D) |

### 用空调的例子理解 P 和 I

**只有 P(比例控制):**
`
目标 26°C
当前 30°C → 差 4°C → 开最大冷气
当前 26.5°C → 差 0.5°C → 稍微开一点冷气
当前 26°C → 不开了

问题:到 26°C 就停,但停了又会回到 26.5°C
      永远差那么一点点 → 这叫稳态误差
`

**加上 I(积分控制):**
`
还差 0.5°C 一直消不掉 → I 慢慢累加这个 0.5
累加到一定程度 → 出力把最后的 0.5°C 打掉
→ 精确到达 26°C
`

### PID 结构体拆解

`c
// pid.h 中的关键变量
typedef struct {
    float Kp;          // 比例系数: 调 P 的力度
    float Ki;          // 积分系数: 调 I 的速度
    float integral;    // 积分累加器: 不停地记着"还没补完"
    float output;      // 最终输出的占空比
    float out_min;     // 输出下限(通常 0.0)
    float out_max;     // 输出上限(通常 0.95)
} PID_t;
`

### 核心运算一步步拆

`c
// pid.c 中的核心函数
float PID_Update(PID_t *pid, float setpoint, float feedback)
{
    // 第1步: 算误差
    // 目标 12V, 实际 11.5V → 误差 = 0.5V
    error = setpoint - feedback;     // = 0.5

    // 第2步: P 项 = 误差 × Kp
    // Kp = 0.5 → P 出力 = 0.5 × 0.5 = 0.25
    // 含义: 占空比增加 0.25
    p_out = Kp * error;

    // 第3步: I 项 = 累积误差 × Ki
    // 上次积分 = 0.1, 这次误差 0.5
    // integral += Ki × 0.5 = 0.01 × 0.5 = 0.005
    // integral = 0.1 + 0.005 = 0.105
    integral += Ki * error;

    // 第4步: 合成输出
    output = p_out + integral;

    // 第5步: 限幅(不能超过 0.95)
    if (output > 0.95) output = 0.95;

    // 结果: 占空比从 0.5 变成了 0.75
    // 电压开始上升 →
    return output;
}
`

### 积分分离:防"上头"

刚开机:目标 12V,实际 0V,误差 = 12V!!! 如果不做处理,积分器疯狂累加 12…24…36…等电压到 12V 时,积分已经积了很大 → 超调到 15V! 然后慢慢降回来。这叫**积分饱和(windup)**。

解决方案:积分分离。如果误差太大(>integral_limit) → 冻结积分,不加了。等电压接近目标了 → 再开始积分。

`c
// pid.c 里的实现
if (fabsf(pid->error) < pid->integral_limit) {
    // 误差在范围内,正常积分
    pid->integral += pid->Ki * pid->error;
} else {
    // 误差太大,冻结积分,不加了
}
`

### 积分限幅:防"积太多"

即使误差不大,积分也可能慢慢积到很大(比如带不动负载,一直差一点)。所以再加一道限幅:

`c
if (pid->integral > pid->out_max) pid->integral = pid->out_max;
`

### 调参手感(怎么调 Kp 和 Ki)

| 现象 | 原因 | 怎么办 |
|------|------|--------|
| 输出突突振荡(高频) | Kp 太大 | 减小 Kp |
| 半天到不了目标 | Kp 太小 | 增大 Kp |
| 输出慢慢晃(低频) | Ki 太大 | 减小 Ki |
| 永远差一点点 | Ki 太小 | 增大 Ki |

**调参四步口诀:**
1. Ki = 0,只调 Kp
2. 增大 Kp 直到输出开始振荡,记下这个值
3. Kp 取这个值的 60%
4. 慢慢加 Ki,直到稳态误差消失;如果振荡了,Ki 减半

### 预置参数模板

`c
// pid.c 底部
const PID_t PID_BUCK_12V_5A = {
    .Kp = 0.5f,
    .Ki = 0.01f,
    // ... 这些是初始值,实际必须根据你的电路整定!
};
`

注意:这些只是**初始猜测值**,不是标准答案。每块板子、每个电感电容都不一样,必须现场调。

### 第三课总结

| 概念 | 人话解释 |
|------|----------|
| P(比例) | "现在差多少,就补多少" |
| I(积分) | "一直差,就一直记着,慢慢补" |
| 积分饱和 | "积太多,收不住,冲过头" |
| 积分分离 | "误差太大时别积了" |
| 限幅 | "占空比不能超过 95%" |

**最简单的记忆**: PID = 看现在差多少 + 记着以前的账 + 猜未来趋势。

## 第四课: 保护模块 — "保险丝和保安"

**对应代码文件: Core/Inc/protection.h + Core/Src/protection.c**

### 为什么保护比控制更重要

电源题最真实的场景:调了三天,终于稳定了。突加负载一瞬间——"啪",MOSFET 炸了。烟冒出来,心凉了,只剩半天时间。

**保护模块就是你的最后一道防线。** 宁可输出暂时不正常,也不能烧东西。

### 过压保护(OVP) — "电压太高了!"

什么情况会过压?
- 反馈环路失控(PI 算错了)
- 负载突然断开(能量没地方去)

后果:输出电容爆浆、负载设备烧毁。

处理:**立即关断,锁死,等人来修。**

### 过流保护(OCP) — "电流太大了!"

什么情况会过流?
- 输出短路(两根线碰一起了)
- 负载太重

后果:MOSFET 过热烧毁、冒烟。

处理:**立即关断,锁死。**

### 欠压保护(UVP) — "输入电压太低了!"

什么情况会欠压?电池快没电了,或者电源线接触不良。

为什么也要保护?输入电压低 → 占空比会变得很大 → 电感可能饱和 → 电感饱和 → 电流失控 → 炸管。

### 锁死机制:为什么不能自动恢复

方案 A:过流了→关断→等 1 秒→自动重启。又过流→关断→又重启→反复炸(每次重启都冲击一次)。

方案 B:过流了→永久关断→需要人按按钮才能恢复。一次故障就停止,等人排查,不会反复损坏硬件。

**电赛里选方案 B。** 宁愿不工作,也别反复炸。

### 代码拆解

`c
void Protection_Check(float vout, float iout, float vin, float temp)
{
    // 一级一级检查
    if (vout > OVP_THRESHOLD) {     // 电压 > 13V?
        Protection_Latch();          // 锁死!
        return;
    }

    if (iout > OCP_THRESHOLD) {     // 电流 > 5.5A?
        Protection_Latch();          // 锁死!
        return;
    }

    if (vin < UVP_THRESHOLD) {      // 电压 < 8V?
        Protection_Latch();          // 锁死!
        return;
    }

    // 全部通过 → 状态正常
    status = PROTECT_NONE;
}
`

### 软启动:温柔地上电

直接上电会发生什么?目标 12V,实际 0V → 误差 12V → PI 全力输出 → 占空比 95% → 输出电容瞬间充电 → 电流巨大 → 过流保护触发。

软启动就是不直接冲到目标,而是"慢慢来":

`
t=0ms:  占空比 0%
t=5ms:  占空比 10%
t=10ms: 占空比 20%
...
t=50ms: 占空比 100% (到达目标)
`

就像你开车不是一脚油门踩到底,而是慢慢加速。

**代码实现:**

`c
float SoftStart_GetDuty(void)
{
    counter++;  // 每次调用计数+1

    // 线性爬升: 当前计数 / 总步数
    duty = counter / SOFT_START_STEPS;
    // SOFT_START_STEPS = 100
    // 第1次: 1/100 = 0.01
    // 第50次: 50/100 = 0.50
    // 第100次: 100/100 = 1.0 (软启动完成)

    return duty;
}
`

在主循环里,软启动阶段的实际参考电压被软启动系数限制:

`c
if (!软启动完成) {
    // 实际参考 = 目标电压 × 软启动百分比
    // 比如 12V × 0.3 = 3.6V → 低速逼近,不冲击
    PID_Update(&pid, target * softstart_duty, vout);
}
`

### 第四课总结

| 概念 | 人话解释 |
|------|----------|
| OVP | 电压太高就断电 |
| OCP | 电流太大就断电 |
| UVP | 输入电压太低也断电 |
| 锁死 | 触发后必须手动复位 |
| 软启动 | 慢慢升压,防止冲击 |

**保护模块的哲学:宁可怂,别硬扛。炸一次管可能就意味着比赛结束。**

## 第五课: main.c — "总指挥部"

**对应代码文件: Core/Src/main.c**

### main.c 在干什么

前面的 4 个模块就像 4 个工具:
- PWM = 手(执行)
- ADC = 眼睛(观察)
- PID = 大脑(决策)
- 保护 = 本能反应(危险就躲)

**main.c 就是总指挥部,安排它们按正确的顺序和时间干活。**

### 程序的完整流程

`
                        上电
                          ↓
                ① 系统初始化
           时钟 → PWM → ADC → PID → 保护
                          ↓
                  ② 启动软启动
              PWM使能,从0%开始爬升
                          ↓
             ┌──── ③ 10kHz 控制中断 ────┐
             │  每 100us:                │
             │  ADC采样 → 保护检查       │
             │  → PID计算 → 更新PWM      │
             └──────────────────────────┘
                          ↓
              ④ while(1): 显示/通信/按键
`

### 什么是中断?为什么这么重要

打个比方:你正在写作业(主循环)。每隔 5 分钟闹钟响一次(中断)。闹钟一响,你停下笔,去看一下电饭锅关了没(执行中断任务)。看完回来继续写作业(回到主循环)。

**中断 = "到时候了!该干正事了!"**

为什么控制要在中断里做?如果放在 while(1) 里:ADC 读数可能被按键打断,PID 计算可能被显示刷新打断 → 控制周期不稳定 → 输出纹波变大。

放在中断里:闹钟一响 → 10kHz 定时 → 雷打不动执行控制 → 控制周期精确 = 输出稳定。

### 控制中断里执行什么

`c
void Control_Loop_ISR(void)  // 每 100us 执行一次
{
    // ① 看: 读取传感器
    ADC_Process();
    vout = ADC_GetVoltage();   // 当前电压
    iout = ADC_GetCurrent();   // 当前电流

    // ② 判断: 有没有危险?
    Protection_Check(vout, iout, vin, temp);
    if (有危险) return;  // 已经切断了,别往下走了

    // ③ 想: 算一下该调多少
    if (软启动中) {
        duty = softstart * 0.5;  // 慢慢爬
    } else {
        duty = PID_Update(&pid, 12V, vout);  // 闭环控制
    }

    // ④ 动: 调水龙头
    PWM_SetDuty(duty);

    // 总共耗时: ~20us (72MHz MCU)
    // 还有 80us 留给其他任务
}
`

### while(1) 里干什么

`c
while (1)  // 无限循环,MCU 永远在跑
{
    // ① 显示电压电流到 OLED
    OLED_Show(adc.vout, adc.iout);

    // ② 检测按键切换电压
    if (KEY1按下) target_voltage = 5V;
    if (KEY2按下) target_voltage = 12V;

    // ③ 串口发数据给电脑调参
    printf("Vout: %.2fV, Iout: %.2fA, Duty: %.3f\r\n",
           vout, iout, duty);

    // ④ 延时 10ms
    HAL_Delay(10);
    // 然后再回来从头执行
}
`

**主循环慢一点没关系。控制的核心在中断里。**

### 完整开发流程

`
第1步: 确认硬件
  □ 焊好功率板(电感、MOSFET、电容、驱动芯片)
  □ 确认辅助电源工作正常
  □ 用万用表量分压电阻值,写入 adc.h

第2步: 开环调试
  □ 烧录代码,手动写死一个占空比(比如 0.4)
  □ 示波器看 PWM 波形 → 确认频率、死区正确
  □ 万用表量输出电压 → 确认分压比算对了
  □ 加电子负载 → 看电压随负载变化

第3步: 闭环调试
  □ Ki = 0,从 Kp = 0.1 开始逐步增加
  □ 直到输出振荡,记下临界 Kp,取 60%
  □ 再加 Ki,直到稳态误差消失
  □ 突加/突卸负载,看恢复速度

第4步: 极限测试
  □ 满载运行 30 分钟 → 看温度
  □ 短路测试 → 保护是否触发
  □ 输入电压上下波动 → 输出是否稳定
`

### 现场排故速查表

| 现象 | 可能原因 | 检查什么 |
|------|----------|----------|
| 没输出 | PWM 没使能 | 示波器量 PWM 引脚 |
| 输出不对 | 分压电阻算错 | 万用表量 ADC 引脚电压 |
| 输出振荡 | PI 参数不对 | 减小 Kp |
| 加载就掉电压 | I 太小或控制频率太低 | 增大 Ki,检查中断频率 |
| 炸管 | 死区不够或占空比太大 | 检查死区时间,DUTY_MAX |
| 纹波大 | 滤波窗口太小 | 增大 ADC_SAMPLE_BUFFER_SIZE |
| 保护频繁触发 | 阈值设太紧 | 适当放宽保护阈值 |
| 发热严重 | 没装散热片或频率不对 | 加散热片,检查开关频率 |
